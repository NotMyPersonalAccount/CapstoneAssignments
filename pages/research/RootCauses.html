Root Causes

Root Cause #1

Open source software is not perceived as being reliable or quality software.

Evidence

People are often only exposed to existing proprietary software such as Windows. [1]

Explantion

People may not otherwise have been exposed to open-source software, and so they might think because they’ve never seen or heard of it and it isn’t widely used, it must be because it’s bad quality.

Evidence

Developers are oftentimes not paid for their contributions. [1]

Explantion

People may think that the quality of work is not good if the developers are not paid, because of the misconceptions that either there might not be as much effort put in or the developers are not qualified.

Implications

People must be shown how good open-source software can be in contrast to proprietary software alternatives so that they may actually consider using it.

Root Cause #2

There is reluctance around transitioning to open-source software.

Evidence

Existing software being used may be using proprietary file formats to store information (i.e documents) that are not well handled by other software. [1]

Explantion

Because the existing proprietary file formats can not be handled well, or at all, migrating may require some time and effort. Additionally, others may continue using the previous file formats, requiring extra effort to share files.

Evidence

People often only know how to use existing proprietary software that they’d learned first. [1]

Explantion

People need to relearn things with the new software, which they stubbornly might not want to do. Additionally, this takes some time and effort and temporarily reduces productivity and efficiency.

Implications

This issue can be solved with more common usage of open standards. Therefore, formats can continue to be used both internally and externally for an organization, and less effort is required to migrate.

References

  1. Petrov, Dmitrij and Obwegeser, Nikolaus, Adoption Barriers of Open-Source Software: A Systematic Review (March 11, 2018).
